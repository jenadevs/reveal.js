cygwin installation

  • make sure to activate “rsync” and “SSH” in the installation
  • add “C:\cygwin64\bin” to your Windows system variable “PATH”

Vagrant Core OS - Linux

install git


$ yum install git

Install openssl-devel and openssl-devel


$ yum install openssl-devel openssl

Install nano


$ yum install nano

Install Vagrant


$ cd
$ mkdir downloads
$ cd downloads
$ cd wget https://dl.bintray.com/mitchellh/vagrant/vagrant_1.7.4_x86_64.rpm
$ rpm -Uvh vagrant_1.7.4_x86_64.rpm

Install VirtualBox


$ vim /etc/yum.repos.d/virtualbox.repo

insert following text


[virtualbox]
name=Oracle Linux / RHEL / CentOS-$relesever / $basearch - VirtualBox
baseurl=http://download.virtualbox.org/virtualbox/rpm/el/$releasever/$basearch
enabled=1
gpgcheck=1
gpgkey=http://download.virtualbox.org/virtualbox/debian/oracle_vbox.asc

save and exit vim

install packages


$ rpm -Uvh http://ftp.joist.ac.jp/pub/Linux/Fedora/epel/7/x86_64/e/epel-release-7-5.noarch.rpm
$ yum install gcc make patch dkms qt libgomp
$ yum install kernel-headers kernel-devel fontforge binutils glibc-headers glibc-devel
$ export KERN_DIR=/usr/src/kernels/2.6.32-504.3.3.el6.x86_64
$ yum install VirtualBox-5.0
$ service vboxdrv setup

Getting started

clone CoreOs-Vagrant repo (win Git Bash)


$ git clone https://github.com/jenadevs/coreos-vagrant.git
$ cd coreos-vagrant
$ git checkout meetup_1_practice

cluster configuration

  • get discovery token from https://discovery.etcd.io/new
  • Edit the file user-data

change


etcd2:
  discovery: https://discovery.etcd.io/NEW_TOKEN

to


etcd2:
  discovery: https://discovery.etcd.io/0850c01e29cb6d7dcd5d4e9123311704

start cluster (Power Shell/ Git Bash on Windows)


$ cd .\git-projects-github\coreos-vagrant
$ vagrant up

if everything works fine, 3 coreos nodes will pop up

stop cluster


$ vagrant halt

destroy cluster


$ vagrant destroy

customizations

increase the count, change 'config.rb' from


# Size of the CoreOs cluster created by Vagrant
$num_instances=3

to this


# Size of the CoreOs cluster created by Vagrant
$num_instances=10

find shared sources here (bug in win 10)


$ cd /home/core/jena-devs/meetup_1

you can place e.g. Dockerfiles here

Start docker container on core os node

login into 1st coreos node


$ vagrant ssh core-01 -- -A

start elastic search container on core os


$ docker run -d elasticsearch

show docker containers


$ docker ps

show docker container IP


$ docker inspect \
	-f '{{.NetworkSettings.Networks.bridge.IPAddress}}' \
	{your_docker_container_id/_container_name}

check connection to elastic search


$ curl '{your_docker_container_IP}:9200/_cat/health?v'

show elastic search nodes


$ curl '{your_docker_container_IP}:9200/_cat/nodes?v'

show elastic search indices


$ curl '{your_docker_container_IP}:9200/_cat/indices?v'

add elastic search index


$ curl -XPUT '{your_docker_container_IP}:9200/meetup_participant?pretty'

post data


$ curl -XPOST '{your_docker_container_IP}:9200/meetup_participant/external?pretty' -d '{
"name":{"first_name":"John","last_name":"Doe"},
"id":3
}'

get data


$ curl '{your_docker_container_IP}:9200/meetup_participant/_search?q=*&pretty'
